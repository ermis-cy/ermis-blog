## Critical Fixes Deployed

Full audit of all layers revealed 12 bugs — 4 critical, 4 high, 4 medium. All fixed and tested.

### Critical: Order Duplication (was losing real money)

The same smart money signal produced **4 duplicate orders** on the same market. Root cause: `match_results.processed` flag was never set to 1 after the decision engine evaluated them. Every 5-minute scan cycle re-evaluated the same signals.

**Fix:** `get_recent_matches()` now filters `WHERE processed = 0`. Decision engine calls `mark_matches_processed()` for ALL evaluated matches after each cycle. SAFETY checks in execution are now **fail-closed** (block trade on exception instead of continuing).

### Critical: Bracket Positions Never Sold

`manage_bracket_positions()` was only updating the DB when take-profit/stop-loss triggered. It never called `_sell_clob_position()`. Tokens stayed in the wallet forever — the 360-379 tweets bracket was up +228% but never cashed out.

**Fix:** Both take-profit and stop-loss paths now call `_sell_clob_position()` before updating DB. Uses actual on-chain balance when share count is unknown.

### Critical: CLOB Minimum Order Size

Polymarket requires minimum **5 shares** per order. Agent was placing $2.55 orders at 54¢ = 4.72 shares → rejected by CLOB. Then the unprocessed signal loop kept retrying every 5 minutes.

**Fix:** Decision engine now rejects bets where `size_usdc < entry_price * 5`. Execution layer enforces `shares < 5.0` as hard floor.

### Critical: Foreign Keys Not Enforced

`PRAGMA foreign_keys=ON` was missing. Trades and positions could reference non-existent markets. No referential integrity.

**Fix:** Added to `connect()`. Existing test caught by the enforcement (fixed to create market first).

### High: Duplicate Open Positions

No constraint prevented multiple open positions on the same market. Race conditions between scan cycles could create duplicates.

**Fix:** Added `CREATE UNIQUE INDEX idx_positions_one_open_per_market ON positions(market_id) WHERE closed_at IS NULL`.

### High: SQL Injection in update_bracket_position

`update_bracket_position(**kwargs)` built SQL from arbitrary caller-supplied column names via f-string.

**Fix:** Whitelist of allowed columns. Raises `ValueError` on disallowed keys.

### High: Wallet Dedup in Smart Money

If one wallet placed 2 buys on the same market, it counted as "2 wallets" — inflating confidence to MEDIUM and displaying "Fernandoinfante, Fernandoinfante".

**Fix:** Dedup by wallet address before computing `n_wallets`, confidence, and display names.

### Medium Fixes

- Added `NOT NULL` on `trades.created_at`, `positions.opened_at`, `match_results.created_at`
- Added `PRIMARY KEY` on `price_snapshots` to prevent duplicate rows
- Added missing indexes: `trades.market_id`, `positions.market_id`, `match_results(processed, created_at)`
- Added `insert_match_results_batch()` for fewer commits at scale
- Added `neg_risk` column migration for markets

### Test Coverage

Added 5 new DB integrity tests (25 total, all passing):
- FK violations raise errors
- Duplicate open positions blocked
- Closed positions allow reopen
- Processed match results filtered
- Column whitelist enforced

Added mandatory pre-commit checklist to CLAUDE.md with 9 audit points. Critical and high severity issues block deploy. No exceptions — this handles real money.
